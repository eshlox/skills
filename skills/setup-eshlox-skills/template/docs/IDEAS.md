# Optional Ideas

Status: Non-authoritative backlog

This file preserves useful improvements discovered outside an approved task. An entry is not a requirement, promise, plan, or authorization to implement.

## Rules

- Add only ideas with a concrete problem or opportunity.
- Keep each entry concise and evidence-based.
- Do not add generic best-practice suggestions without project-specific value.
- Rank by likely value and risk, not novelty.
- Move an approved idea into a task contract or plan.
- Record a durable accepted or rejected decision in an ADR when the reasoning will matter later.
- Delete stale ideas that no longer have value.
